Frontend
----------
1.The part of a website that the user interacts with directly is called frontend.
2.In frontend it is also referred as client side of the application.
3.It includes everything that user experience directly.
4.HTML,CSS,JS,BootStrap are the languages used for frontend development.

Backend
--------
1.Backend is the server side of the website.
2.It stores and arranges the data also make sure everything on the client side of
  the web site works fine.
3.In backend there are some languages like java,python,php,.NET...etc.

API
----
1.It means application programming interface.
2.Which is collection of communication protocols used by various programs to communicate between them.

Storages Areas
---------------
1.As part of our applications we required to store data like customers information or billing information,
  calls information...etc.

2. To store this data we required storage areas.

There are 2 types of storage areas such as
	1) Temporary Storage Areas
	2) Permanent Storage Areas
1)Temporary Storage Areas
--------------------------
These are the memory areas where the data will be stored temporally.
Eg:-all JVM memory areas (like Heap Area, Stack Area, Method Area, PC Registers, Native Method stacks).
    Once JVM Shutdown all these memory areas will cleared automatically.
2)Permanent Storage Areas
-------------------------
Also known as persistent storage areas. Here we can store data permanently.
Eg:- Files systems, Databases, Data warehouse, Bigdata...etc.

File Management System(FMS)
----------------------------
DEF:-1. A file management system is a system where data is stored in files on the operation systems,
---
      each application program must handle its own storage data, retrieval and updating.
     2.File systems can be provided by the local operating system.

     3. File system are best suitable for to store very less amount of information(like KB)
Eg:- A library management system storing books in separate text files likes, books.txt, members.txt

Limitations of File Management Systems(FMS)
--------------------------------------------
1. Data redundancy :- same data is stored in multiple files.
2. Data Inconsistency :- updates in one file may not be reflected in another.
3. Poor Data Security :- No proper accessed control.
4. Difficult Data Retrieval :- searching requires custom programs.
5. Integrity issues :- No constraints (Eg:- roll number  uniqueness)
6. Scability issues :- Hard to manage as data grows.

To over come the above problems of file systems, we should go for databases.

Database Management System
---------------------------
Def:- A DBMS is a collection of programs that enables users to create, manage, and manipulate database.

It acts as an interface between user and database.

Advantages
-----------
1.We can store huge amount of information in databases.
2.Query language support is available for every database and hence we can perform database operations very easily.
3.To accesses Data present in database, compulsory username and password must be required. Hence data is secured.
4.Inside database data will be stored in the form of tables. While developing database table schemas, database admin
  fallow various normalization techniques and we can implement various constraints like Unique key constraint, primary
  key constraints, ... etc. which prevent data duplication. Hence there is no chance of data inconsistency problems.

Limitation of Databases
------------------------
1) Database cannot store huge amount of information like terabytes(1000 GB) of data.
2) Database can provide support for only structured data (tabular data or relational data)
   and cannot provide support for semi structured data like (xml files) and unstructured data
   (like video files, audio files, images... etc)

To over come these problems we should go for advanced storage areas like big data, data warehouse etc.

Q) What are the differences between File Management System(FMS) and Database Management System(DBMS) ?
---------------------------------------------------------------------------------------------------------
1. Data is stored in separate filed
   Data is stored in structured data
2. High data redundancy(duplicate data)
   Reduce data redundancy
3. Difficult to maintain and update
   Easy to maintain and update
4. Security is limited
   Provides strong security mechanisms.
5. Data sharing is difficult
   Multiple user can access data simultaneously.
6. No relationship between tables.
   Supports relationship between tables.
7. Backup and recovery are difficult.
   Provides backup and recovery features.
8. Data inconsistency many occur anomalies.
   Maintains data consistency and integrity.
9. Suitable for small applications.
   Suitable for small, medium, and large applications.

Q) What is the difference between a file system and a DBMS ? (Important Question)
----------------------------------------------------------------------------------
A file system stores data in separate files and may lead to data redundancy, and security
issues, A DBMS stores data in structured manner using tables, reduces redundancy, provides
security, supports multiple users, and ensures data consistency and integrity.
