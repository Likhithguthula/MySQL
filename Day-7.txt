Constraints
------------
Constraints are rules applied to a table columns in MySQL to ensure that only valid and acurate data is stored in the database.


Why do we use Constraints ?
----------------------------
1. Maintain Data Integrity
2. To prevent invalid data entry
3. Avoid duplicate records
4. Ensures relations between tables
5. Make the database reliable and constraint.

Types of Constraints
---------------------
1. Not Null 
2. Unique key : It doesn't alow duplicate values
3. Primary Key : It is the combination of unique + Not Null
4. Composite key
5. Foreign key
6. Check constraint
7. Default constraint

Foreign Key Constraint
-----------------------
1) A foreign key is a key used to link two tables together.
2) A foreign key is a field or (collection of fields) in one table that refers to the primary key in another table.
3) The table containing the Foreign key is called child table and the table containing the candidate key is called referenced table or parent table.

What are referential Actions
-----------------------------
1) Referential actions define what happens to child table data when parent table data is updated or deleted.
2) They are used with foreign key constraints;

1. On Delete Cascade :- When a parent table row is deleted, all matching child rows are automatically deleted.
2. On Update Cascade :- When a parent key is updated, all matching child foreign key are updated automatically.
